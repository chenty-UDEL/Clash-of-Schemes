# 如何查看构建错误

## 方法一：Vercel 部署日志（推荐）

### 步骤：

1. **访问 Vercel Dashboard**
   - 打开 https://vercel.com
   - 登录你的账号
   - 找到项目 `clash-of-schemes`

2. **查看部署日志**
   - 点击项目进入详情页
   - 点击 **"Deployments"** 标签
   - 找到最新的部署（通常是最上面的）
   - 点击部署条目
   - 查看 **"Logs"** 或 **"Build Logs"** 标签

3. **复制错误信息**
   - 在日志中找到错误信息
   - 复制完整的错误信息
   - 粘贴给我，我就能帮你修复

### 示例错误格式：

```
Failed to compile.

./types/game.ts:17:16
Type error: Cannot find name 'BoardType'.
```

---

## 方法二：本地构建测试

### 步骤：

1. **在项目目录中运行构建命令**
   ```powershell
   cd C:\权谋决战\quan-mou-jue-zhan-full
   npm run build
   ```

2. **查看错误输出**
   - 构建过程中的所有错误都会显示在终端
   - 复制错误信息给我

3. **类型检查**
   ```powershell
   npm run type-check
   ```
   这会只检查类型错误，不进行完整构建

---

## 方法三：VS Code 错误提示

### 步骤：

1. **打开项目**
   - 在 VS Code 中打开项目文件夹

2. **查看问题面板**
   - 按 `Ctrl+Shift+M` 打开问题面板
   - 或者点击底部状态栏的 "Problems"

3. **查看错误**
   - 所有 TypeScript 和 ESLint 错误都会显示
   - 点击错误可以跳转到对应文件

```powershell

# 2. ESLint 检查
npm run lint

# 3. 完整构建
npm run build

# 4. 开发模式（会显示实时错误）
npm run dev
```

---

## 常见错误类型

### 1. TypeScript 类型错误
- 格式：`Type error: ...`
- 解决：修复类型定义或类型导入

### 2. 导入错误
- 格式：`Cannot find module '...'`
- 解决：检查导入路径和文件是否存在

### 3. 语法错误
- 格式：`Unexpected token ...`
- 解决：检查语法是否正确

### 4. 配置错误
- 格式：`Invalid next.config.ts options`
- 解决：检查配置文件
