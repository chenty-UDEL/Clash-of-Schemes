# 🚀 快速部署指南

## 📋 部署步骤

### 第一步：提交代码到 Git

在项目目录中运行：

```powershell
cd C:\权谋决战\quan-mou-jue-zhan-full

# 1. 查看修改的文件
git status

# 2. 添加所有修改的文件
git add .

# 3. 提交（写清楚提交信息）
git commit -m "feat: 添加角色配置系统和基础API路由"

# 4. 推送到 GitHub
git push
```

### 第二步：Vercel 自动部署

推送代码后，Vercel 会：
- ✅ 自动检测到新的提交
- ✅ 自动触发新的部署
- ✅ 自动构建和部署

**无需手动操作！**

---

## 🔍 查看部署状态

### 方式一：Vercel Dashboard

1. 访问 https://vercel.com
2. 登录你的账号
3. 找到你的项目 `clash-of-schemes`
4. 点击进入项目
5. 在 **"Deployments"** 标签中查看部署状态

### 方式二：GitHub

1. 访问你的 GitHub 仓库
2. 在 **"Actions"** 标签中可以看到 Vercel 的部署状态

---

## ⚡ 手动触发部署（如果需要）

如果自动部署没有触发，可以手动触发：

1. 在 Vercel Dashboard 中
2. 点击 **"Deployments"**
3. 点击右上角的 **"Redeploy"** 按钮
4. 选择要重新部署的版本

---

## ✅ 部署完成检查

部署成功后：

1. **查看部署 URL**
   - 在 Vercel Dashboard 中会显示部署 URL
   - 例如：`clash-of-schemes.vercel.app`

2. **访问网站**
   - 打开部署 URL
   - 检查页面是否正常加载

3. **检查构建日志**
   - 在 Vercel Dashboard → Deployments → 点击最新部署
   - 查看 **"Logs"** 确认没有错误

---

## 🆘 如果部署失败

### 检查构建日志

1. 在 Vercel Dashboard 中
2. 点击失败的部署
3. 查看 **"Logs"** 标签
4. 找到错误信息

### 常见错误

1. **TypeScript 类型错误**
   - 修复类型错误
   - 重新提交

2. **缺少环境变量**
   - 检查 Vercel 项目设置中的环境变量
   - 确保所有变量都已配置

3. **依赖安装失败**
   - 检查 `package.json`
   - 确保所有依赖都正确

---

## 📝 部署检查清单

在部署前确认：

- [ ] 环境变量已在 Vercel 中配置
- [ ] 没有 TypeScript 错误
- [ ] 没有构建错误
